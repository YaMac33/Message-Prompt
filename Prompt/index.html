② 記事一覧（カードグリッド）ページ用プロンプト（index.html）

※あなたのstyle.cssには .project-list / .project-card が既にあるので、それを活かす版。

あなたはフロントエンド実装に厳密なエンジニアです。以下の条件で、完成した index.html（記事一覧ページ）だけを出力してください。

【サイト構成・前提】 - /ai-comparison-test-site/header.html と /ai-comparison-test-site/style.css が既にある。 - ヘッダーは header.html をフェッチして <div id="header-mount"></div> に挿入（GitHub Pages、SSI不可）。 - 追加のCSSは禁止（必要最小限のインラインCSSのみ20行以内）。外部フレームワーク不可。

【ページ要件】 - DOCTYPEから閉じタグまでフルの index.html。 - /ai-comparison-test-site/style.css を <head> で1回だけ読み込む（重複禁止）。 - 固定ヘッダー考慮。本文は上部余白を確保。 - メインに<h1>と、.project-list（CSSのグリッドを流用）を持つカード一覧を配置。 - 各カード（.project-card）は a要素で全体クリック可能、タイトル（h3）、説明（p）、投稿日（time datetime属性付き）、タグ（ul/li）を含む。 - 10〜30件のダミーデータをJSの配列から生成し、DOMで .project-list に差し込む方式にする（将来JSON差し替え想定）。 - フィルタ/検索UIを上部に設置（入力でタイトル・タグに対して絞り込み）。 - 並び替えのセレクタを用意（新着順/古い順/タイトルA→Z）。 - IntersectionObserverでカードのフェードイン。カードhoverで持ち上がり（既存CSSの transform を活用）。 - 右下に「トップへ戻る」ボタン。 - すべてのJSにコメントを入れる。 - Google Analyticsは header.html にあるため重複禁止。

【実装詳細（重要）】 - ヘッダー読み込みは fetch('/ai-comparison-test-site/header.html') → DOMParser で <header> を抽出し #header-mount に挿入。 - 検索は入力の変化で即時フィルタ（toLowerCase、全角半角・カタカナ/ひらがな差は今回は未対応でOK）。 - ソートはArray.sortで実装。日付は 'YYYY-MM-DD' をDateに変換して比較。 - カードクリックで記事URLへ遷移（/ai-comparison-test-site/posts/slug/ を想定）。 - アクセシビリティ：入力にlabel、カードにaria-label、画像があればalt。

【ページタイトル】 - 「記事一覧 | SmartSideNote」

【ダミーデータ（例）】 - title, slug, description, date (YYYY-MM-DD), tags (配列), optional: thumbnail
